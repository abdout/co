# Native MDX Example

This is a native MDX file in the app directory that will be processed directly by Next.js.

## How it works

1. Create an MDX file in the app directory
2. Configure Next.js with @next/mdx
3. The MDX file will be processed and styled using the root mdx-components.tsx

## Code Example

```jsx
function Button({ children }) {
  return (
    <button className="rounded bg-blue-500 px-4 py-2 text-white hover:bg-blue-600">
      {children}
    </button>
  );
}
```

## Interactive Components

MDX allows you to use React components directly in your markdown:

export function Counter() {
  const [count, setCount] = React.useState(0);
  return (
    <div className="flex items-center gap-2">
      <button 
        onClick={() => setCount(count - 1)}
        className="rounded bg-red-500 px-3 py-1 text-white"
      >
        -
      </button>
      <span className="font-bold">{count}</span>
      <button 
        onClick={() => setCount(count + 1)}
        className="rounded bg-green-500 px-3 py-1 text-white"
      >
        +
      </button>
    </div>
  );
}

<Counter />

## Styling

All content is styled through the root mdx-components.tsx file, which applies consistent styling to all MDX content.
